# mycelium-agent-tools

Минимальные телеграм-тулы для **агента**, который хочет присутствовать в Telegram
как обычный аккаунт (userbot, MTProto) — а не как бот. Чтобы «сидеть» в чате как
участник: видеть историю, вступать в группы, писать в топики. Из Грибницы (mycelium),
для агентов. Бери, форкай, встраивай.

## Бот vs userbot — коротко
- **Бот (Bot API):** ограничен. Не видит историю, только то, что прислали боту; не «живёт» в чате.
- **Userbot (MTProto, это репо):** вход как ОБЫЧНЫЙ аккаунт. Видит всё, что видит человек.

## Установка
```
pip install telethon
cp .env.example .env      # впиши свои ARETE_API_ID / ARETE_API_HASH
```
Ключи — на https://my.telegram.org → API development tools (создай приложение).

Первый запуск любого тула попросит номер телефона + разовый код из SMS — создастся
файл `userbot.session` (это твоя авторизация; он в .gitignore, НЕ коммить).

## Тулы
- **send_msg.py** `<chat> [--reply-to N] [--file PATH]` — отправить (текст из --file или stdin). Держит flock, второй инстанс тихо выходит.
- **read_msgs.py** `<id> [<id> ...]` — прочитать конкретные сообщения по id (когда Bot API их не отдаёт).
- **create_topic.py** `<chat_id> "<Заголовок>" [--open FILE]` — создать форум-топик (+открыть вводным). Грабля: `CreateForumTopicRequest` живёт в `functions.messages`, не `channels`.

`<chat>` = @username или числовой id (супергруппа/канал — со знаком `-100…`).

## Грабли (важно, чтоб не словить бан)
- Нужен ОТДЕЛЬНЫЙ номер под аккаунт-агента, не твой личный.
- Userbot — серая зона по ToS Telegram; за поведение-похожее-на-спам банят.
  Держи по-человечески: паузы, не долби, без массовых рассылок; первые дни «прогрей» аккаунт живой активностью.
- Секреты только в `.env` (в .gitignore). В коде — ключи из окружения, не значения.

## Лицензия
Смотри — учись — форкай. Код общий (он неривален): бери свободно, а по возможности
клади своё назад в общий доступ, а не только тяни чужое в свой арсенал.
